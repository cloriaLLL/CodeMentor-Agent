## 1. 文档概述

本规范定义 CodeMentor Agent 的三大核心 Agent：
- **Orchestrator**（教学主控）— 状态机驱动，管理 Teach/Practice/Ecosystem 三态流转
- **Generator**（出题）— 基于 seed_data 生成场景化 Mini-Project
- **Validator**（校验）— 静默预跑参考答案，确保零坏题率

### 1.1 设计原则（匹配 最终成品设计.md §1.1）
1. **节拍驱动 (Rhythm-Driven)**：Orchestrator 主动推演，非被动响应
2. **严格基准 (Strict Grounding)**：依赖官方 PEP/Docs，杜绝幻觉
3. **零坏题率 (Zero-Broken Guarantee)**：Generator-Validator 双 Agent 预跑机制
4. **Prompt 模块化分离**：严禁硬编码在业务逻辑中

### 1.2 目录结构
```
codementor-mvp/
├── agents/
│   ├── __init__.py
│   ├── orchestrator.py          # Orchestrator 状态机实现
│   ├── generator.py             # Generator Agent 实现
│   └── validator.py             # Validator Agent + Retry Loop 实现
└── prompts/
    ├── orchestrator.txt         # Orchestrator System Prompt
    ├── generator.txt            # Generator System Prompt
    └── validator.txt            # Validator System Prompt
```

## 2. 三大 Agent 职责定义

### 2.1 Orchestrator Agent（教学主控）
**职责来源**：最终成品设计.md §2.1

- 管理当前学习会话状态（`TeachMode` / `PracticeMode` / `EcosystemMode`）
- 读取并更新用户画像上下文（`learner_profile`），控制节点推演图谱
- 调用 RAG 知识元，输出概念 + 版本演进 + 工业最小代码示例
- 不直接调用 LLM 生成内容（MVP 阶段从 seed_data 读取预置内容）

### 2.2 Generator Agent（出题）
**职责来源**：最终成品设计.md §2.1 + Demo设计.md §2.2 接口 2

- 基于 AST 提取的开源种子代码，按当前节点生成场景化 Mini-Project 描述与单元测试
- 输出 JSON 结构：`problem_statement` + `starter_code` + `reference_solution` + `test_cases`
- MVP 阶段：基于 `seed_data.json` 的种子题库，结合 Prompt 生成变体

### 2.3 Validator Agent（校验）
**职责来源**：最终成品设计.md §2.1 + Demo设计.md §2.2 接口 2

- 在题目推送到前端前，静默调用沙盒预跑参考答案
- 不通过则捕获 Trace 触发内部重试 Loop（上限 3 次）
- 检查测试用例覆盖边界场景、starter_code 无逻辑泄露

## 4. Prompt 文件加载机制

### 4.1 加载函数

```python
# agents/__init__.py 或 agents/prompt_loader.py
from pathlib import Path

PROMPTS_DIR = Path(__file__).parent.parent / "prompts"

def load_prompt(name: str) -> str:
    """
    从 prompts/ 目录加载 System Prompt
    :param name: prompt 名称（不含扩展名），如 "orchestrator"
    :return: prompt 全文
    """
    prompt_path = PROMPTS_DIR / f"{name}.txt"
    if not prompt_path.exists():
        raise FileNotFoundError(f"Prompt file not found: {prompt_path}")
    return prompt_path.read_text(encoding="utf-8")
```

```python
        ...
```

**禁止行为：**
- ❌ 在业务代码中硬编码 System Prompt 字符串

## 5. Orchestrator 状态机实现

### 5.1 状态枚举

```python
# agents/orchestrator.py
from enum import Enum

class LearningState(str, Enum):
    TEACH = "TeachMode"
    PRACTICE = "PracticeMode"
    ECOSYSTEM = "EcosystemMode"
```

### 5.2 状态流转规则

```
[初始] → TeachMode
TeachMode --(action="go_to_practice")--> PracticeMode
PracticeMode --(submit passed=true)--> EcosystemMode
PracticeMode --(submit passed=false)--> PracticeMode (重试)
EcosystemMode --(next_node_recommendation)--> TeachMode (新节点)
```

### 5.3 接口契约

```python
class OrchestratorAgent:
    def __init__(self):
        self.system_prompt = load_prompt("orchestrator")  # 预留给 LLM 调用
        self.seed_data = self._load_seed_data()
    
    def _load_seed_data(self) -> dict:
        """加载预置数据（匹配 Demo设计.md §1.1 预置 JSON 元数据）"""
        seed_path = Path(__file__).parent.parent / "schemas" / "seed_data.json"
        return json.loads(seed_path.read_text(encoding="utf-8"))
    
    def teach(self, node_id: str) -> TeachContent:
        """
        State 1: 从 seed_data 读取知识元，返回教学内容
        所有内容从数据文件读取，禁止硬编码
        """
        atom = self._find_atom(node_id)
        return TeachContent(
            markdown_content=atom["teach_content"]["markdown_content"],
            grounding_source=atom["grounding_doc"]["source_name"],
            history_notes=atom["evolution_history"]["key_changes"],
            next_actions=atom["teach_content"]["next_actions"],
        )
    
    def transition_to_practice(self, node_id: str) -> None:
        """状态流转到 PracticeMode（仅状态切换，无内容生成）"""
        ...
    
    def ecosystem_summary(self, node_id: str) -> EcosystemContent:
        """
        State 3: 从 seed_data 读取生态映射，返回生态总结
        stack_summary 从 ecosystem_mapping.stack_summary 读取，禁止硬编码
        """
        atom = self._find_atom(node_id)
        eco = atom["ecosystem_mapping"]
        return EcosystemContent(
            stack_summary=eco["stack_summary"],
            cross_language_equivalent=eco["cross_language_equivalents"],
            next_node_recommendation="python.advanced.asyncio",  # 固定推荐下一节点
        )
    
    def _find_atom(self, node_id: str) -> dict:
        """从 seed_data 查找指定知识元"""
        atom = next(
            (a for a in self.seed_data["knowledge_atoms"]
             if a["node_id"] == node_id),
            None
        )
        if atom is None:
            raise ValueError(f"Node not found: {node_id}")
        return atom
```

**关键约束（重要）：**
- ❌ **禁止**在 `agents/orchestrator.py` 中硬编码任何教学内容或生态总结文本
- ✅ **必须**从 `schemas/seed_data.json` 的 `knowledge_atoms[].teach_content` 和 `knowledge_atoms[].ecosystem_mapping` 读取
- ✅ 生产环境可调用 LLM 基于 `prompts/orchestrator.txt` 动态生成（替换预置数据读取）

## 6. Generator-Validator Retry Loop 流程

### 6.1 流程图

```
┌─────────────────────────────────────────────────────────┐
│  /api/generate_exercise 入口                            │
└──────────────────────┬──────────────────────────────────┘
                       │
                       ▼
              ┌─────────────────┐
              │  attempt = 0    │
              └────────┬────────┘
                       │
                       ▼
        ┌──────────────────────────────┐
        │  Loop: while attempt < 3    │
        └──────────────┬──────────────┘
                       │
                       ▼
    ┌──────────────────────────────────────┐
    │ 1. Generator.generate(node_id)      │
    │    → 生成 problem/solution/tests     │
    └──────────────┬──────────────────────┘
                   │
                   ▼
    ┌──────────────────────────────────────┐
    │ 2. Validator.validate(solution,      │
    │       test_cases)                    │
    │    → 调用 sandbox.run_solution_tests │
    └──────────────┬──────────────────────┘
                   │
                   ▼
              ┌────────────┐
              │ status?    │
              └─────┬──────┘
                    │
        ┌───────────┼───────────┐
        │           │           │
        ▼           ▼           ▼
   VALIDATED   REJECTED    EXCEPTION
        │           │           │
        │           │           │
        ▼           ▼           ▼
   返回成功     attempt++    attempt++
   给前端       (捕获错误)    (捕获 Trace)
                    │           │
                    └─────┬─────┘
                          │
                          ▼
                   attempt >= 3?
                   ├── 是 → 返回 422
                   │         (retry exhausted)
                   └── 否 → 继续 Loop
```

### 6.2 Retry Loop 代码契约

```python
# agents/validator.py
from sandbox import run_solution_tests

MAX_RETRY = 3  # 严格匹配 Demo设计.md §2.2 接口 2: "上限 3 次"

class ValidatorAgent:
    def __init__(self):
        self.system_prompt = load_prompt("validator")
    
    def validate(self, exercise: GeneratedExercise) -> ValidationResult:
        """
        静默预跑参考答案
        :return: ValidationResult(status, comments, execution_result)
        """
        result = run_solution_tests(
            solution_code=exercise.reference_solution,
            test_cases_code=exercise.test_cases,
        )
        if result.status == "passed":
            return ValidationResult(
                status="VALIDATED",
                comments=f"All {result.total_count} tests passed",
                execution_result=result,
            )
        else:
            return ValidationResult(
                status="REJECTED",
                reason=f"{result.failed_count} failed, {result.error_count} errors",
                execution_result=result,
            )

def generate_with_retry(node_id: str) -> GeneratedExercise | None:
    """
    Generator-Validator Retry Loop
    :return: 通过校验的 exercise，或 None（重试耗尽）
    """
    generator = GeneratorAgent()
    validator = ValidatorAgent()
    
    for attempt in range(MAX_RETRY):
        exercise = generator.generate(node_id)
        result = validator.validate(exercise)
        
        if result.status == "VALIDATED":
            return exercise
        
        # 捕获 Traceback 并反馈给 Generator（下一轮重试时参考）
        generator.feedback(result.reason, result.execution_result)
    
    return None  # 重试耗尽
```

### 6.3 与 main.py 的集成（匹配 DOC-01 §5.2）

```python
# main.py
from agents.validator import generate_with_retry

@app.post("/api/generate_exercise")
async def api_generate_exercise(req: GenerateExerciseRequest):
    exercise = generate_with_retry(req.node_id)
    
    if exercise is None:
        return JSONResponse(
            status_code=422,
            content={
                "status": "error",
                "message": "Generator-Validator retry exhausted (3 attempts)",
                "code": 422,
            },
        )
    
    return GenerateExerciseResponse(
        status="success",
        exercise_id=exercise.exercise_id,
        problem_statement=exercise.problem_statement,
        starter_code=exercise.starter_code,
        validator_status="PASSED_ZERO_BROKEN",
    )
```

## 7. Generator Agent MVP 实现策略

### 7.1 MVP 简化方案

Demo设计.md §2.2 接口 2 描述「调用 Generator Agent 基于 seed_data.json 结合 Prompt 生成题目」。

**MVP 阶段实现策略：**
- **基础策略**：直接从 `seed_data.json` 读取预置的 `seed_problem` 全量字段（含 `problem_statement`、`code_skeleton`、`reference_solution`、`test_cases`），所有内容均为预置数据，**禁止在 Python 代码中硬编码任何题目说明或假问答文本**
- **增强策略（可选）**：调用 LLM 基于 `seed_problem` + `generator.txt` Prompt 生成变体题目（生产环境方案）

```python
# agents/generator.py
import json
from pathlib import Path

SEED_DATA_PATH = Path(__file__).parent.parent / "schemas" / "seed_data.json"

class GeneratorAgent:
    def __init__(self):
        self.system_prompt = load_prompt("generator")  # 预留给 LLM 调用
        self.seed_data = self._load_seed_data()
    
    def _load_seed_data(self) -> dict:
        return json.loads(SEED_DATA_PATH.read_text(encoding="utf-8"))
    
    def generate(self, node_id: str) -> GeneratedExercise:
        """
        MVP: 从 seed_data 加载预置题目（全部字段从数据文件读取，不硬编码）
        生产环境：调用 LLM 基于 system_prompt + seed 生成变体
        """
        seed = next(
            (s for s in self.seed_data["seed_problems"]
             if s["associated_node_id"] == node_id),
            None
        )
        if seed is None:
            raise ValueError(f"No seed problem found for node: {node_id}")
        
        # 所有字段从 seed_data.json 读取，不在代码中硬编码任何内容
        return GeneratedExercise(
            exercise_id=seed["seed_id"].replace("seed_", "ex_"),
            problem_statement=seed["problem_statement"],      # 从数据文件读取
            starter_code=seed["code_skeleton"],               # 从数据文件读取
            reference_solution=seed["reference_solution"],     # 从数据文件读取
            test_cases=seed["test_cases"],                     # 从数据文件读取
        )
    
    def feedback(self, reason: str, execution_result) -> None:
        """
        接收 Validator 反馈，用于下一轮重试
        MVP: 无 LLM 调用，重试无效（直接返回 None）
        生产: 将失败原因注入 Prompt，调用 LLM 重新生成
        """
        pass  # MVP 阶段无 LLM 调用
```

**关键约束（重要）：**
- ❌ **禁止**在 `agents/generator.py` 中硬编码任何 `problem_statement` 文本
- ❌ **禁止**在 Python 代码中拼接假的题目说明 Markdown
- ✅ **必须**从 `schemas/seed_data.json` 读取所有题目内容（符合 Demo设计.md §1.1 预置 JSON 数据）
- ✅ 生产环境通过 LLM + `prompts/generator.txt` 动态生成（替换预置数据读取）

## 8. Validator Agent 实现契约

### 8.1 数据结构

```python
from pydantic import BaseModel
from typing import Literal
from sandbox import ExecutionResult

class ValidationResult(BaseModel):
    status: Literal["VALIDATED", "REJECTED"]
    comments: str | None = None
    reason: str | None = None
    execution_result: ExecutionResult | None = None

class GeneratedExercise(BaseModel):
    exercise_id: str
    problem_statement: str
    starter_code: str
    reference_solution: str
    test_cases: str
```

### 8.2 Validator 实现要点

- 调用 `sandbox.run_solution_tests()` 执行 `reference_solution + test_cases`
- 解析 `ExecutionResult.status`：
  - `passed` → `VALIDATED`
  - `failed` / `error` / `timeout` → `REJECTED`
- 检查 starter_code 无逻辑泄露（AST 检查）

```python
import ast

def check_starter_code_safety(starter_code: str) -> bool:
    """AST 检查 starter_code 仅包含骨架，无逻辑泄露"""
    tree = ast.parse(starter_code)
    for node in ast.walk(tree):
        # 检查函数体仅包含 pass 或 docstring
        if isinstance(node, ast.FunctionDef):
            body = node.body
            for stmt in body:
                if isinstance(stmt, ast.Pass):
                    continue
                if isinstance(stmt, ast.Expr) and isinstance(stmt.value, ast.Constant):
                    continue  # docstring
                if isinstance(stmt, ast.AnnAssign):  # type hint
                    continue
                return False  # 发现逻辑实现
    return True
```

## 9. 错误处理

### 9.1 Agent 层异常

```python
class AgentError(Exception):
    """Agent 基础异常"""

class GeneratorError(AgentError):
    """Generator 生成失败"""

class ValidatorError(AgentError):
    """Validator 校验失败"""

class RetryExhaustedError(AgentError):
    """重试耗尽"""
```

### 9.2 与 main.py 错误响应的映射

| 异常 | HTTP Code | message |
|------|-----------|---------|
| `GeneratorError` | 500 | "Exercise generation failed" |
| `ValidatorError` | 500 | "Validation failed" |
| `RetryExhaustedError` | 422 | "Generator-Validator retry exhausted" |
| `ValueError`（节点不存在） | 404 | "Node not found" |
