## 1. MVP 目标与裁剪边界 (Scope & Target)

[cite_start]为了在短期内实现流畅的 Showcase 闭环，系统在保持深度技术架构的前提下，将展示范围固定为 Python 黄金闭环链路 [cite: 18, 41, 63]：

* [cite_start]**知识库**：采用预置 Markdown 文件及 JSON 元数据，替代全网实时爬虫 [cite: 11, 38, 57]。

## 2. MVP 系统架构与 API 契约 Specification

### 2.1 目录结构规范
```text
codementor-mvp/
├── main.py                     # FastAPI 后端服务入口与路由定义
├── sandbox.py                  # 代码隔离执行与 pytest 测试结果解析器
├── schemas/
│   └── seed_data.json          # 预置装饰器知识元与种子题库 JSON
├── static/
│   └── index.html              # 左侧 Chat + 右侧 Monaco Editor 双栏前端 UI
└── app/requirements.txt        # 项目依赖 (fastapi, uvicorn, pytest, requests)

```

### 2.2 API RESTful 接口契约

#### 接口 1: 教学对话与知识讲解 (State 1: TeachMode)

* **Endpoint**: `POST /api/teach`
* **Request Payload**:
```json
{
  "node_id": "python.advanced.decorator",
  "action": "start"
}

```


* **Response Payload**:
```json
{
  "status": "success",
  "state": "TeachMode",
  "markdown_content": "# Python 装饰器与高阶函数\n...",
  "grounding_source": "PEP 318 -- Decorators for Functions and Methods",
  "history_notes": "Python 2.4 正式引入 @ 语法糖...",
  "next_actions": ["go_to_practice", "more_examples"]
}

```



#### 接口 2: 双 Agent 实战出题 (State 2: PracticeMode)

* **Endpoint**: `POST /api/generate_exercise`
* **Request Payload**:
```json
{
  "node_id": "python.advanced.decorator"
}

```


* **Processing Logic**:
1. 调用 Generator Agent 基于 `seed_data.json` 结合 Prompt 生成题目说明、Starter Code、Solution Code 与 Test Cases 。


2. 调用 Validator Agent，将 Solution Code 与 Test Cases 写入临时目录并调用 `sandbox.py` 执行 。


3. 若通过，返回生成结果；若失败，捕获 Traceback 并自动 Retry (上限 3 次) 。




* **Response Payload**:
```json
{
  "status": "success",
  "exercise_id": "ex_decorator_rate_limit",
  "problem_statement": "请实现带参数的限流装饰器 @rate_limit(max_calls=3, period=60)...",
  "starter_code": "def rate_limit(max_calls: int, period: int):\n    # 请补全代码\n    pass\n",
  "validator_status": "PASSED_ZERO_BROKEN"
}

```



#### 接口 3: 用户代码提交与判题

* **Endpoint**: `POST /api/submit_code`
* **Request Payload**:
```json
{
  "exercise_id": "ex_decorator_rate_limit",
  "user_code": "import time\nfrom functools import wraps\n..."
}

```


* **Response Payload**:
```json
{
  "status": "success",
  "passed": true,
  "score": 100,
  "pytest_output": "3 passed in 0.02s",
  "next_state": "EcosystemMode"
}

```



#### 接口 4: 生态收尾与总结 (State 3: EcosystemMode)

* **Endpoint**: `POST /api/ecosystem_summary`
* **Request Payload**:
```json
{
  "node_id": "python.advanced.decorator"
}

```


* **Response Payload**:
```json
{
  "status": "success",
  "state": "EcosystemMode",
  "stack_summary": "在 FastAPI + Redis 架构中，装饰器常作为路由中间件与限流器使用...",
  "cross_language_equivalent": {
    "Go": "Gin 框架 Middleware (HandlerFunc 链)"
  },
  "next_node_recommendation": "python.advanced.asyncio"
}

## 3. 核心 Agent System Prompts 规范

### 3.1 教学 Orchestrator System Prompt

```text
You are CodeMentor-Orchestrator, an AI tutor designed around the "Teach -> Practice -> Apply" framework.

[CONSTRAINTS]
1. Strict Grounding: Rely strictly on the provided PEP 318 context. Never hallucinate nonexistent syntax.
2. Structure: Always present output in 3 parts:
   - Core Concept Definition
   - Historical Evolution Context (e.g., Python 2.4 @ introduction)
   - Industrial Minimal Code Example
3. Concise Tone: Clear, encouraging, highly structured for learning.

```

### 3.2 出题 Generator System Prompt

```text
You are CodeMentor-Generator. Your task is to output a scenario-based mini-project programming problem.

[INPUT]
Target Topic: Python Decorators
Seed Scenario: Redis/In-memory Rate Limiting Middleware

[OUTPUT FORMAT REQUIREMENT]
Must output valid JSON matching:
{
  "problem_statement": "Markdown text describing the task",
  "starter_code": "Python code skeleton for user",
  "reference_solution": "Complete correct implementation",
  "test_cases": "Complete pytest assertions"
}

```

### 3.3 校验 Validator System Prompt

```text
You are CodeMentor-Validator. You inspect the output generated by Generator.

[TASK]
1. Take 'reference_solution' and 'test_cases'.
2. Check if the test assertions cover edge cases.
3. Verify if 'starter_code' contains no logic leaks (should only have skeleton and pass/type hints).
4. Output JSON: {"status": "VALIDATED", "comments": "..."} or {"status": "REJECTED", "reason": "..."}.

```
