# CodeMentor Agent — 对话驱动的 AI 编程教学智能体

![Python](https://img.shields.io/badge/Python-3.12-blue) ![FastAPI](https://img.shields.io/badge/FastAPI-0.115-teal) ![React](https://img.shields.io/badge/React-19-61dafb) ![TypeScript](https://img.shields.io/badge/TypeScript-6-blue) ![GLM-4](https://img.shields.io/badge/LLM-GLM--4--Flash-green) ![License](https://img.shields.io/badge/License-MIT-lightgrey)

> **"让每一位编程学习者都拥有一位懂节奏、会出题、能判题的私人 AI 导师。"**

CodeMentor Agent 是一个面向编程学习者的 **AI 智能教学平台**。它以独创的"三层教学结构 + 四步教学节奏"为核心教学法，将 AI 对话从无序的"一问一答"升级为有节奏的结构化教学，并内置多语言安全沙箱实现"学完即练、提交即判"的完整闭环。

**大赛主题**：智驱未来，Agent 创新 · **开发平台**：火山引擎 Trae

> 📖 本文为比赛面向的**项目总介绍**。本地运行与开发说明见 [开发指南](../README.md)。

## 📖 项目简介

### 我们在解决什么问题？

当前编程教育存在一个普遍痛点——**学习资源碎片化、学习路径不可控、练习反馈缺失闭环**：

| 现状 | 痛点 |
|------|------|
| 文档站点（MDN / 菜鸟教程） | 只提供静态知识，无法互动练习 |
| 刷题平台（LeetCode / 牛客） | 只提供题目，不提供系统性教学 |
| AI 助手（ChatGPT / Copilot） | 能答疑但缺乏结构化节奏，容易"一问一答"式碎片化 |

学习者往往陷入"看懂文档 → 不会写代码 → 刷题受挫 → 放弃"的恶性循环，缺少一个能将 **"学 → 练 → 用"** 串联成标准化节拍的智能导师。

### 我们的解决方案

CodeMentor Agent 将 **AI 对话教学 + 多语言代码沙箱 + 学习进度持久化** 三者融为一体：

```
  学（AI 结构化教学）  →  练（沙箱即时判题）  →  用（知识点迁移推进）
         ↑                      ↓                      ↓
         └──── 进度持久化 ←──── 反馈闭环 ←──── 掌握度评估
```

## 🎯 核心创新

### 1. 三层四步教学法

传统 AI 教学是"用户问什么就答什么"，没有节奏控制。CodeMentor Agent 创新了 **三层递进 + 四步固定节奏** 的教学法：

```
┌─────────────────────────────────────────────────────────────┐
│  第一层：对话层（感知）                                       │
│  用户发起学习意向 → Agent 给出 200 字简短概况 → 引导深入       │
└──────────────────────────┬──────────────────────────────────┘
                           │ 用户选择"深入"
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  第二层：教学层（理解 → 应用）                                │
│  Step 1  实例先行  — 完整可运行代码，先看效果                  │
│  Step 2  概念跟进  — 定义 + 语法 + 版本差异                   │
│  Step 3  溯源深化  — 设计哲学 + 历史演进 + 常见陷阱            │
│  Step 4  练习巩固  — 场景化题目，学完即练                     │
└──────────────────────────┬──────────────────────────────────┘
                           │ 四步完成
                           ▼
┌─────────────────────────────────────────────────────────────┐
│  第三层：推进层（迁移）                                       │
│  Agent 推荐后续知识点 → 用户选择继续 / 停下 / 复习             │
└─────────────────────────────────────────────────────────────┘
```

**为什么有效**：实例先行降低认知门槛 → 概念跟进建立知识框架 → 溯源深化避免"知其然不知其所以然" → 练习巩固形成肌肉记忆。

### 2. Agent 驱动的智能体架构

系统由三个专业 Agent 协同工作，各司其职：

| Agent | 职责 | 技术亮点 |
|-------|------|----------|
| **OrchestratorAgent**（主控） | 控制教学节奏与状态机流转 | 三层四步状态机 + 上下文感知 |
| **ExerciseGeneratorAgent**（出题） | 根据知识点生成 4 种题型练习 | LLM 结构化输出 + JSON Schema 校验 |
| **ExerciseEvaluator**（评估） | 沙箱执行判题 + AI 智能反馈 | 代码质量评估 + 个性化教学反馈 |

### 3. 多语言安全沙箱

内置支持 **5 种编程语言** 的安全执行引擎，学完即练、提交即判：

| 语言 | 运行方式 | 测试框架 |
|------|----------|----------|
| Python | 直接执行 | pytest |
| JavaScript | Node.js | exit-code 判定 |
| Bash | Shell | exit-code 判定 |
| Java | javac + java | exit-code 判定 |
| C# | dotnet-script | exit-code 判定 |

**安全防护**：5 层纵深防御（输入验证 → AST 白名单 → forbidden 模式匹配 → 资源限制 → 执行隔离）；执行隔离采用 **Docker 容器隔离首选 + 进程树隔离自动降级** 双模式，安装 Docker 即升级为容器级隔离，无 Docker 时恒可用进程树隔离兜底，确保用户代码不会危害系统。

### 4. 三级 LLM 降级策略

```
智谱 GLM-4-Flash（永久免费，推荐）  ──失败──▶  Ollama（本地离线）  ──失败──▶  Mock（兜底可用）
```

- **零成本启动**：智谱 GLM-4-Flash 系列永久免费，无 Token 上限
- **离线可用**：支持 Ollama 本地模型，完全断网也能教学
- **永不宕机**：Mock 兜底保证服务始终可启动，降低部署门槛

## ✨ 核心功能

### 📚 结构化教学
- 三层递进式教学引导，AI 主动控场而非被动应答
- 四步固定节奏（实例 → 概念 → 溯源 → 练习），不跳步不乱序
- 跨语言等价物对比 + 工业生态总结，帮助知识迁移

### 🧪 即时代码练习
- **4 种题型**：理解题（选择/填空）、修改题（基于已有代码改写）、创作题（从零编写）、项目题（综合应用）
- 沙箱即时执行 + pytest 自动判题
- AI 个性化反馈：答错时不只说"错了"，而是当场讲解知识点
- **智能代码编辑器**：多语言语法高亮 + 基于 MiniLang 编译器的语言服务（智能补全、实时诊断/错误提示）

### 📓 笔记本模式
- 章节式知识管理，支持父-子对话结构
- 按知识点组织学习记录，形成清晰的知识结构
- 支持跨章节回顾与复习

### 💾 学习进度持久化
- SQLite 存储学习会话、知识点掌握度、练习提交记录
- 支持断点续学，关闭后重新打开继续上次进度
- 学习路径可回溯，为后续个性化推荐积累数据

### 🎨 毛玻璃模糊效果 UI
- React 19 + Tailwind CSS v4 现代化界面
- 双栏分屏工作区：左侧对话教学 + 右侧代码练习
- SSE 流式输出：逐字显示 AI 回复，实时反馈
- 键盘快捷键：`Ctrl+N` 新建对话、`Ctrl+Shift+N` 新建笔记本、`Ctrl+B` 切换侧边栏
- PWA 支持：可安装为桌面应用

## 🎬 使用方法

### 典型学习流程

```
1. 启动应用 → 打开浏览器访问 http://localhost:8000

2. 发起学习
   └── 在对话框输入："我想学 Python 装饰器"
       └── AI 返回 200 字概况 + "是否深入？" 引导

3. 深入学习（四步教学）
   ├── Step 1: AI 展示完整装饰器示例代码
   ├── Step 2: AI 讲解装饰器定义、语法、版本差异
   ├── Step 3: AI 深入设计哲学、历史演进、常见陷阱
   └── Step 4: AI 出题（理解题/修改题/创作题/项目题）

4. 练习与反馈
   ├── 理解题：选择答案 → AI 判定对错 + 讲解
   ├── 修改题：在示例代码上修改 → 沙箱运行测试用例
   ├── 创作题：从零编写代码 → 沙箱运行 + 代码质量评分
   └── 项目题：综合应用 → 多维度评估

5. 推进学习
   └── AI 推荐下一个知识点（如：闭包、高阶函数）
       └── 用户选择：继续学习 / 停下休息 / 复习已学
```

### 快速开始

#### 环境要求

- Python 3.12+
- Node.js 18.0+

#### 后端启动

```bash
# 1. 创建并激活虚拟环境
python -m venv .venv
.venv\Scripts\activate                   # Windows
# source .venv/bin/activate              # Mac/Linux

# 2. 安装依赖
pip install -r app/requirements.txt

# 3. 配置环境变量
copy .env.example .env                   # Windows
# cp .env.example .env                   # Mac/Linux
# 编辑 .env 填入 ZHIPU_API_KEY（注册地址：https://bigmodel.cn，永久免费）

# 4. 启动服务
python main.py                           # 默认 8000 端口
```

> 💡 **uv 用户**：也可使用 [uv](https://docs.astral.sh/uv/) 包管理器一步完成（无需手动创建虚拟环境）：
> ```bash
> uv pip install -r app/requirements.txt && uv run python main.py
> ```

#### 前端启动（开发模式）

```bash
cd frontend
npm install
npm run dev                              # 5173 端口，自动代理 /api 到后端
```

#### 生产部署

```bash
cd frontend && npm run build             # 构建前端到 dist/
python main.py                           # 后端自动托管前端，访问 http://localhost:8000
```

> 💡 **零成本体验**：智谱 GLM-4-Flash 模型永久免费、无 Token 上限，注册即可使用。也可使用 Ollama 完全离线运行。

> 📖 完整的环境配置、开发命令与测试说明详见 [开发指南](../README.md)。

## ⚠️ 已知局限

以下局限**不影响核心教学闭环**，但值得在使用前了解：

| 局限 | 说明 | 影响范围 | 规划 |
|------|------|----------|------|
| **LLM 提供商单一** | 目前深度适配智谱 GLM-4-Flash，其他 OpenAI 兼容 API（如 DeepSeek、Moonshot）未充分测试 | 不影响核心教学功能，智谱免费模型已覆盖绝大多数场景 | 中期计划接入多提供商 |
| **非 Python 语言测试能力有限** | 沙箱对 Python 支持 pytest 自动判题，其他 4 种语言仅支持退出码判定（通过/失败） | 不影响代码运行和基础判题，但无法展示具体测试用例通过情况 | 计划接入各语言原生测试框架 |
| **默认进程树隔离** | 沙箱支持 Docker 容器隔离（首选，需本地安装 Docker）与进程树隔离（自动降级，恒可用）双模式 | 默认无 Docker 环境下为进程级隔离，适合教学场景；安装 Docker 后自动升级为容器级隔离（网络隔离 + 资源限制） | 已实现双模式，未来可扩展更多隔离后端 |
| **单机部署** | 使用 SQLite 单文件存储，未支持多实例水平扩展 | 不影响单机使用，但无法支撑大规模并发用户 | 长期计划支持 PostgreSQL + Redis |
| **无用户认证系统** | 当前为单用户模式，无登录/权限管理 | 适合个人使用和比赛演示，不适合多用户共享部署 | 中期计划添加用户系统 |
| **知识图谱预设** | 知识点基于 `seed_data.json` 预设，主要覆盖 Python 生态 | 不影响已有知识点的教学质量，但知识覆盖范围有限 | 计划支持运行时动态扩展 |
| **界面仅中文** | UI 文案和 prompt 均为中文 | 不影响中文用户使用，但不支持国际化 | 长期计划支持多语言 |
| **LLM 首字响应延迟** | 教学对话与题目生成依赖 LLM API，首字响应可能需要 5-15 秒（取决于模型和网络） | 不影响功能正确性；已通过 SSE 流式输出（逐字显示）+ "正在思考…"动画缓解等待感 | 后续支持本地小模型加速进一步降低延迟 |

> 💡 以上局限均为**工程完善度**问题，而非**架构设计**缺陷。三层四步教学法、Agent 协同、沙箱判题等核心能力已完整实现并验证。

## 🌐 部署说明

### 项目运行方式

本项目为 **AI 编程教学智能体**，采用 FastAPI + React 前后端一体化架构，内置多语言安全沙箱（Python / JavaScript / Bash / Java / C#）。核心功能已完整实现并通过测试，**本地运行稳定**。

**快速启动**：

```bash
# 后端
pip install -r app/requirements.txt
python main.py              # 默认 http://localhost:8000

# 前端（开发模式）
cd frontend && npm install && npm run dev
```

### 未提供在线演示链接的原因

本项目的技术特性决定了其**不适合部署在静态托管平台**（如 Vercel / GitHub Pages）：

| 技术特性 | 静态托管平台的限制 |
|---|---|
| **多语言沙箱执行用户代码**（子进程 + Docker 容器隔离） | Serverless 函数无法 spawn 任意子进程，无 Docker daemon |
| **SQLite 文件数据库**（学习进度持久化） | Serverless 文件系统临时只读，重启即丢失 |
| **SSE 流式输出 + LLM 长时调用**（5-15 秒） | 平台函数超时受限，流式长连接受限 |
| **FastAPI 常驻服务** | 静态托管不支持长驻 Python 进程 |

因此，项目未提供在线演示链接。评委可通过本地运行体验完整功能。项目代码结构完整、文档详实，核心创新（三层四步教学法 + Agent 驱动 + 多语言沙箱）均已实现并验证。
